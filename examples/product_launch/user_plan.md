# Example User Plan: Product Launch

## Profile

- Project label: Solo SaaS beta launch
- Timezone: America/New_York
- Morning planning time: 08:30
- Evening check-in time: 22:30
- Weekly review time: Sunday 19:00
- Reminder style: direct
- Default focused-time target: 3h baseline + 2h stretch
- Minimum acceptable focused time: 90 minutes
- Output preferences:
  - Interactive HTML workbench: yes
  - Desktop wallpaper PNG: yes
  - Markdown report: yes

## North Star

Launch a usable private beta for a small SaaS product before the demo deadline.

## Phase Plan

### Phase 1: Build beta spine

- Dates: Month 1
- Primary outcome: a demoable core workflow with onboarding, one main user action, persistence, and basic error handling.
- Required outcomes:
  - Core user flow works locally and in staging.
  - Payment and admin work are explicitly deferred.
  - A short demo script exists.
  - Open bugs are tracked and ranked.
- Gate dates:
  - End of Month 1: demoable beta spine exists.
- Work that must stay bounded:
  - Landing page polish.
  - Settings screens.
  - Analytics dashboards.

### Phase 2: Private beta hardening

- Dates: Month 2
- Primary outcome: invite 5-10 testers and fix the highest-friction issues.
- Required outcomes:
  - Tester onboarding notes.
  - Error logging.
  - Top 10 bug list.
  - Beta feedback summary.

## Current Month Plan

- Month: Month 1
- Monthly outcomes:
  - Finish the main user flow.
  - Deploy staging.
  - Prepare a 5-minute demo script.
  - Keep marketing polish capped.
- Active workstreams:
  - Core flow implementation.
  - Staging deploy.
  - Demo script.
  - Bug triage.
- Active micro-sprints:
  - Core flow finish: 7 days.
  - Demo script: 3 days.
- Priority rules:
  - Every weekday includes one visible core-flow move until the demo path works.
  - Marketing polish is capped at 30 minutes unless the demo path is already working.
  - New feature ideas go to backlog unless they remove a demo blocker.
- Known deadlines:
  - Demo review at the end of Month 1.

## Reporting Preferences

- Focused work includes coding, debugging, user-flow design, writing demo script, and reviewing agent output.
- Delegated agent work includes test generation, copy drafts, bug reproduction, and refactor suggestions.
- Distractions to call out directly: landing page tweaking, color changes, adding settings before core flow works.
