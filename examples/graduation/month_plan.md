# Example Month Plan: Month 1

## Monthly Outcome

Recover schedule clarity, close Manuscript A, and make the core analysis chapter draftable.

## Active Workstreams

1. Manuscript A closure
   - Status: about 90 percent complete.
   - Month target: submission-ready, submitted, coauthor-ready, or externally blocked.

2. Core analysis framework
   - Status: code refactor underway.
   - Month target: reviewable state and at least one validated analysis path.

3. Dataset workflow
   - Status: workflow optimization and testing underway.
   - Month target: document status, requirements, tests, and limitations.

4. Optional modeling preparation
   - Status: early preparation only.
   - Month target: short scope note only unless core work is protected.

5. Secondary framework projects
   - Status: scaffold-only.
   - Month target: minimal setup after thesis-critical work is safe.

## Active Micro-Sprints

| Micro-sprint | Window | Target |
| --- | --- | --- |
| Manuscript A exit | 5 days | Move Manuscript A out of daily workload. |
| Core chapter gate | Month 1 | Make the core chapter draftable with main path and figure slots. |

## Priority Rules

- If Manuscript A is not closed, every weekday includes one closure action unless a stronger core blocker exists.
- Core analysis and dataset workflow are the default primary tracks after Manuscript A closure.
- Optional modeling receives only bounded planning until the core chapter is protected.
- Secondary framework projects remain capped unless they directly support the dissertation core.
